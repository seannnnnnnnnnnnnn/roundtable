# 多 Agent 圆桌讨论

## 一个问题，不只听一种答案

把文案、选题、销售话术、产品方案或商业想法交给一桌立场不同的 AI。

Roundtable 会先补齐真正影响判断的条件，再组织多个独立角色展开交锋，最后把分歧压缩成一份清晰、可执行的辩证回答。

**[立即下载 macOS 版](https://github.com/seannnnnnnnnnnnnn/Agent-Roundtable/releases/latest)**

---

## 你得到的不是“支持”或“反对”

普通 AI 往往沿着你的提问继续回答，很容易放大原有判断。Roundtable 会主动引入不同目标、利益和风险视角，让一个想法先经受质疑，再进入行动。

每次圆桌最终都会告诉你：

- 这个想法在什么条件下成立
- 支持它最有价值的理由是什么
- 哪个风险最不能忽略
- 双方真正争议的前提是什么
- 下一步应该保留什么、修改什么、怎么执行

## 为什么它更适合重要问题

### 先补条件，再下判断

系统不会拿到一句话就急着回答，而是用少量选择题补齐受众、目标、身份、场景等关键条件。

### 让不同立场真正交锋

每个角色都有独立的判断标准。正反观点先分别形成，再交叉回应，不是一个 AI 在同一段话里自问自答。

### 保留冲突，不强行中庸

不同价值目标会被明确展示。你能看到每条路线为什么成立，以及选择它需要承担什么代价。

### 10 秒先看结论

默认结果被压缩成一句话判断、成立条件、最大风险、关键分歧和建议改法。需要时再展开角色立场与讨论过程。

### 角色由你决定

开桌前可以选择参与角色、调整角色立场和判断标准，让同一个问题接受真正不同的审视。

## 适合这些场景

- **内容决策**：选题、标题、图文内容、短视频脚本是否值得发布
- **销售转化**：话术是否有说服力，哪里容易引起误解或流失
- **产品判断**：功能、定价、定位和用户需求是否真正成立
- **商业想法**：机会、风险、隐藏前提和执行成本是否匹配
- **团队讨论**：把模糊争论变成可见分歧和下一步行动

## 一次圆桌如何进行

1. 输入一个需要判断的问题或方案
2. 回答最多三个关键条件问题
3. 选择并确认本轮讨论角色
4. 等待角色独立判断和交叉回应
5. 阅读压缩结论，按需继续深挖风险、改得更稳或生成可用版本

## 下载与安装

前往 **[最新版本下载页](https://github.com/seannnnnnnnnnnnnn/Agent-Roundtable/releases/latest)**：

- Apple 芯片 Mac（M1/M2/M3/M4）：选择 `arm64`
- Intel Mac：选择 `x64`

下载 DMG 后，将 Roundtable 拖入“应用程序”。

当前社区版本首次打开时，如被 macOS 提醒，请在 Finder 中右键 Roundtable，选择“打开”。

## 开始使用

打开左下角“设置”，填写兼容 OpenAI API 的地址、模型名称和 API Key，即可新建第一场圆桌。

API Key、项目、会话和圆桌结果均保存在你的 Mac 本机，不会提交到本仓库。
